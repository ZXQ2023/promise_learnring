# Promise 库源码解读

这一模块用来拆解那些大量依赖 Promise 编排异步流程的优秀开源库。前面的章节已经从零实现了 Promise；这里换一个角度：看真实项目如何把 Promise 用在请求调度、任务排队、错误传递、取消控制、插件机制这些更复杂的场景里。

阅读源码时会重点关注三件事：

- **Promise 链路**：异步任务如何被串起来，成功和失败如何沿链条传递。
- **状态边界**：库在什么位置把同步逻辑切到异步逻辑，又在哪里把错误统一包装。
- **工程抽象**：拦截器、适配器、取消机制这类能力如何围绕 Promise 组合出来。

## 建议读法

不要一上来就逐行啃源码。先找出「调用入口」「Promise 链」「真正执行异步任务的位置」，再回头补配置合并、环境适配、边界处理等细节。这样源码会从一团线慢慢变成一条清楚的河道。
